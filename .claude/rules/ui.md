# UI / UX Guideline（要約）

方針：UIは「要素を増やさず、迷わせず、失敗させない」を最優先にする。
  - MUST optimize for clarity and error-prevention.

## 上位10（常時守る）
1. シンプルにする（要素を増やさない）
2. 簡単にする（手順を減らす）
3. メンタルモデルに合わせる
4. シグニファイア（押せそう/押せなさそうを明確に）
5. 一貫性（同じものは同じ表現・挙動）
6. ユーザーの言葉を使う（専門用語を避ける）
7. ユーザーの記憶に頼らない（その場で参照できる）
8. エラーを回避する（起きない設計 + 建設的メッセージ）
9. 可逆性（戻せる/取り消せる）
10. メジャータスク最適化（80%に合わせる）
